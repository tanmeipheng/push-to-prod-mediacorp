- Added `logging` setup so tenacity can log retry warnings and info messages.
- Introduced `is_rate_limit_error(exception)` predicate that returns `True` only when the raised `HTTPError` carries an HTTP 429 status code, ensuring other HTTP errors are not retried.
- Decorated `sync_data()` with `@retry` configured with:
  - `retry=retry_if_exception(is_rate_limit_error)` — retries only on 429 errors.
  - `wait=wait_exponential(multiplier=1, min=2, max=60)` — exponential backoff starting at 2 s, capped at 60 s.
  - `stop=stop_after_attempt(6)` — gives up after 6 total attempts.
  - `reraise=True` — re-raises the original exception after exhausting retries.
  - `before_sleep` / `after` log hooks for observability.
- Original function signature and docstring are fully preserved.
- Test file covers: success after two 429s, immediate success, exhausted retries raising HTTPError, no retry on 500, and single-retry success.
